# WolfCV :: Trust And Sharing v0

## Status

Pencil document.

This document exists because the next phase is not just “make it better”.

It is:

```text
make it shareable without turning it into dishonest product theater
```

---

## 1. The sharing problem

A machine can work internally
and still fail as a shared tool.

Why:

- users do not know what to trust
- users do not know what counts as failure
- AI products often hide uncertainty behind polished text

`WolfCV` cannot afford that.

Its whole point is trust.

---

## 2. Trust surfaces that must stay visible

The user must be able to see:

- what the target vacancy really looks like
- what the repo truth actually supports
- what the machine translated
- where the machine distrusted itself
- where the reading was degraded

If any of these become invisible,
product sharing starts to corrupt the project.

---

## 3. What should never be hidden

Do not hide:

- unsupported claims
- degraded vacancy reads
- weak evidence
- guard discomfort
- role mismatch
- missing domain experience

The product should become easier.
It should not become softer in its truth boundaries.

---

## 4. First sharing posture

The first people who should see `WolfCV` are:

- technically strong peers
- repo-shaped candidates
- machine-literate reviewers

Not broad anonymous users yet.

This matters because the earliest sharing phase is:

- feedback collection
- interpretation debugging
- trust calibration

not mass onboarding.

---

## 5. Human-readable explanation

For sharing to work,
the user needs a short explanation that says:

- what WolfCV does
- what it does not do
- what inputs it expects
- what outputs matter first
- why degraded outputs are not bugs to hide

This should become a future human-facing quickstart or README layer.

---

## 6. Trust law

The product must allow this sentence to remain true:

```text
WolfCV may translate truth.
It may not replace truth.
```

That is the line that should survive every polish pass.

---

## 7. Short formula

```text
sharing begins when another human can run the tool,
read the outputs,
and tell what is strong, weak, or missing
without being deceived by smooth wording.
```
